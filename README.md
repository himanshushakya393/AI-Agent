This project is a Conversational AI Agent built for a fictional SaaS product called AutoStream. The idea behind this project is to simulate how a real SaaS chatbot works — answering user queries, identifying interested users, and capturing leads.

The agent can:

Understand what the user is asking (intent detection)
Answer pricing and policy questions using a local knowledge base
Detect when a user is interested in buying (high intent)
Collect user details and store them using a mock function
▶️ How to Run the Project Locally
Clone the repository:
git clone <your-repo-link>
cd AI_agent
Create a virtual environment:
python -m venv venv
Activate it (Windows):
venv\Scripts\activate
Install required libraries:
pip install -r requirements.txt
Run the project:
python app.py

After running, the chatbot will start in the terminal and you can interact with it.

🧠 Architecture Explanation

For this project, I used LangGraph because it helps in designing structured workflows for conversational agents. Instead of writing everything in one file, I divided the logic into steps like intent detection, response generation, and tool execution.

The agent works in a simple flow — first it detects the user’s intent, then based on that it either answers using the knowledge base or starts collecting user details if the user shows high interest.

To manage conversation flow, I used a shared state object. This state stores things like user input, detected intent, and user details such as name, email, and platform. Because of this, the agent can remember previous responses and continue the conversation smoothly across multiple turns.

I also made sure that the lead capture function is only called after all required details are collected. This prevents incorrect or early execution.

📱 WhatsApp Integration (Webhook Explanation)

To integrate this agent with WhatsApp, I would use the Meta WhatsApp Cloud API.

First, I would create a webhook using Flask or FastAPI. This webhook will receive messages whenever a user sends a message on WhatsApp.

When a message comes in, I would extract the user’s message and pass it to the agent. The agent will process the message, detect intent, and generate a response.

To keep track of conversations, I would store user data (like name, email, etc.) in a database such as Redis or Firebase. This helps in maintaining conversation flow across multiple messages.

Finally, the response generated by the agent is sent back to the user through the WhatsApp API.

Flow:
WhatsApp → Webhook → Agent → Response → WhatsApp API
